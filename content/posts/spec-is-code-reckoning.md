## Research Focus

The agentic coding movement's promise and its fundamental critique

## The Central Claim

The agentic coding movement promises to replace code with specifications. The critique is devastating:

> If you make a spec precise enough to reliably generate working code, you've just written code in a worse format.

## Dijkstra's Narrow Interfaces

You can't escape the fundamental difficulty of capturing edge cases, error conditions, and system interactions. You can only transmute that labor into prose.

- **Code:** Explicit, testable, executable
- **Specs:** Ambiguous, interpretable, requires translation

## The Engineering Parallel to Consciousness

This critique mirrors the consciousness problem:
- **Specs:** Exhibit code-like behavior without being code
- **AI:** Exhibits belief-like behavior without necessarily having belief

In both cases, we're pretending that abstraction layers solve fundamental difficulties. They don't — they just hide them.

## The Meta-Pattern

A spec that doesn't capture edge cases is worse than code that fails explicitly. A model that exhibits belief-like behavior may not have belief.

The deepest insights come from questioning the frameworks themselves, not optimizing within them.
